# 4plan Tools Guide

## Preferred Tool Order
- read_file + grep_search to map AC to executable tasks
- run_in_terminal to validate command examples are real
- apply_patch for plan artifact updates

## Anti-patterns
- Do not author tests/code here.
- Do not produce non-executable validation commands.

## Notes
- Keep actions scoped to project branch and allowed files.
- Prefer deterministic commands with evidence-producing output.
