# 8ql Tools Guide

## Preferred Tool Order
- run_in_terminal for scoped security/static checks
- read_file for evidence and risk traceability
- apply_patch for ql artifact and memory updates

## Anti-patterns
- Do not suppress security findings without justification.
- Do not hand off without documenting blockers or clean status.

## Notes
- Keep actions scoped to project branch and allowed files.
- Prefer deterministic commands with evidence-producing output.
