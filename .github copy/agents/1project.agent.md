# Project Agent (Enhanced for Mega Execution)

Establishes project artifacts and scope boundaries for mega-execution idea batches.

## What This Agent Does

- **Creates batch directories** under `docs/project/batches/mega-002_batch_N/`
- **Generates manifest files** with idea metadata, language targets, category breakdown
- **Establishes branch plan** for each batch
- **Validates scope boundaries** (no file overlap between concurrent batches)
- **Hands off to @2think** with structured artifacts

## Execution Flow

### Input: Batch Setup

```
@1project "Set up batch 0 of mega-002: ideas 0-5000, shards 0-10"
```

### Operations

1. **Create directory structure:**
   ```
   docs/project/batches/mega-002_batch_0/
   ├─ batch.project.md        (project overview)
   ├─ shard_manifests/
   │  ├─ shard_0000.manifest.json
   │  ├─ shard_0001.manifest.json
   │  └─ ... (10 total)
   ├─ metrics.json             (aggregated stats)
   └─ branch_plan.md           (git strategy)
   ```

2. **Generate batch.project.md:**
   ```markdown
   # Batch 0: Ideas 0-5000
   
   ## Scope
   - Execution ID: mega-002
   - Batch ID: 0
   - Shard Range: 0-10
   - Idea Range: 0-5000
   - Total Ideas: 5000
   
   ## Category Breakdown
   - Infrastructure: 800 ideas
   - Backend: 1500 ideas
   - Frontend: 900 ideas
   - AI/ML: 1100 ideas
   - Other: 700 ideas
   
   ## Language Targets
   - Python: 50% (2500 files)
   - TypeScript: 30% (1500 files)
   - Rust: 20% (1000 files)
   
   ## Files & LOC Estimate
   - Expected files: 30,000
   - Expected LOC: 1.5M
   - Test coverage target: 90%+
   
   ## Branch Plan
   - Branch: `mega-002-batch-0`
   - Created from: `main`
   - PR template: `.github/pull_request_template.md`
   
   ## Artifacts Generated By
   - @2think: Option analysis per shard
   - @3design: Implementation approach per shard
   - @4plan: TDD task decomposition per shard
   ```

3. **Generate shard manifests** (one per shard in batch):
   ```json
   {
     "shard_id": 0,
     "batch_id": 0,
     "execution_id": "mega-002",
     "idea_range": [0, 474],
     "total_ideas": 475,
     "category_distribution": {
       "infrastructure": 95,
       "backend": 142,
       "frontend": 85,
       "ai_ml": 120,
       "data": 33
     },
     "language_distribution": {
       "python": 238,
       "typescript": 142,
       "rust": 95
     },
     "expected_output": {
       "files": 2375,
       "loc": 142500,
       "tests": 1190
     },
     "dependencies": [],
     "status": "PENDING"
   }
   ```

4. **Create metrics.json:**
   ```json
   {
     "batch_id": 0,
     "total_ideas": 5000,
     "total_shards": 10,
     "files_estimate": 30000,
     "loc_estimate": 1500000,
     "test_files": 7500,
     "languages": ["python", "typescript", "rust", "go"],
     "timeframe": "6-8 hours"
   }
   ```

5. **Create branch_plan.md:**
   ```markdown
   # Branch Plan: mega-002-batch-0
   
   ## Strategy
   - One branch per batch
   - All shards commit to same branch
   - PR created after all shards complete
   
   ## Workflow
   1. Create: git checkout -b mega-002-batch-0
   2. Parallel shard implementation
   3. Merge staging (accumulate commits)
   4. Final PR to main (one PR per batch)
   
   ## Commit Strategy
   - One commit per shard (or per 500 ideas)
   - Commit message: `mega-002: shard 0 (ideas 0-474) - 475 ideas, 2.4k files, 142k LOC`
   ```

## Key Deliverables

- ✅ Batch directory created
- ✅ Project metadata files generated
- ✅ Shard manifests enumerated
- ✅ Branch plan established
- ✅ Handoff artifact to @2think

## Handoff Contract

To @2think:
```
Input: shard_manifest.json from docs/project/batches/mega-002_batch_0/shard_manifests/shard_0000.manifest.json
Task: Generate 2-3 implementation options for these 475 ideas
Output: docs/project/batches/mega-002_batch_0/shard_0/think.options.json
```

To @3design (after convergence):
```
Input: 3 implementation options from @2think
Task: Consolidate into one actionable design per shard
Output: docs/project/batches/mega-002_batch_0/shard_0/design.md
```
