# prj0000027 — Encrypted Memory Blocks: Git History

## Branch
`prj0000027-encrypted-memory-blocks`

## What Was Done
Implemented and validated the Rust-backed encrypted memory block subsystem.

### Rust implementation (`rust_core/src/memory.rs`)
- `EncryptedMemoryBlock` — stores slabs as `nonce || ciphertext` (ChaCha20-Poly1305)
- `MemoryBlockRegistry` — thread-safe `DashMap`-backed registry; each registry owns a `StaticSecret`
- `derive_dek()` — X25519 ECDH → HKDF-SHA-256 key derivation (deterministic, per-block DEK)
- `PyMemoryBlockRegistry` — PyO3 wrapper registered in `memory::register()`

### Rust deps (all already in Cargo.toml)
- `chacha20poly1305 = "0.10"`, `x25519-dalek = "2"`, `hkdf = "0.12"`, `dashmap = "6"`,
  `zeroize = "1"`, `uuid = "1"`, `subtle = "2"`

### Tests created
- `tests/test_encrypted_memory.py` — 12 Python tests covering:
  - Block creation (UUID format)
  - Slab index increment
  - Encrypt/decrypt round-trip (text and binary)
  - `slab_count()`, `purge()`, `remove_block()`
  - Registry isolation: separate registries cannot share blocks
  - OOB slab index error
  - Empty payload round-trip

### Rust unit tests (in `rust_core/src/memory.rs`)
- DEK determinism
- DEK uniqueness per ephemeral key
- Full encrypt/decrypt round-trip at the Rust level
