# prj0000074 — workspace-meta-improvements

## Goal
Improve the overall workspace quality, agent workflow completeness, and documentation coherence across five distinct sub-goals:

1. **Agent file co-location** — Move `docs/agents/*.memory.md` and `docs/agents/*.log.md` to `.github/agents/` so all agent-related files (definitions, memory, logs) live in one place. Keep naming conventions uniform.

2. **Docstrings policy** — Update `.github/agents/5test.agent.md` and `.github/agents/6code.agent.md` to require that every function they write or modify must include a Google-style docstring. Add enforcement via a flake8/ruff rule (`D` codes).

3. **Architecture documentation** — Reduce `docs/architecture/` to a readable, maintainable set of files. Remove generated bloat (`Architecture.generated.md`, `todolist.md`). Rename files to a uniform namespace: `0overview.md`, `1agents.md`, `2backend.md`, `3rust.md`, `4web.md`, `5data.md`. Ensure the agent workflow keeps these up to date (add a step to @6code.agent.md to update the relevant arch doc after each code change).

4. **Kanban SWOT + risk management** — Expand `docs/project/kanban.md` with SWOT analysis and risk-management sections: a risk register table (ID, description, likelihood, impact, mitigation) and a SWOT table per active project or globally.

5. **Deeper @2think** — Expand `.github/agents/2think.agent.md` to include: more research tasks (literature review, alternative enumeration, prior-art search), structured output format, and a minimum time/depth guideline so @2think doesn't jump to conclusions too quickly.

## Scope boundary
- Agent definition files: `.github/agents/*.agent.md` (update instructions only — no code)
- Agent memory/log files: `docs/agents/` → `.github/agents/` (move + update references)
- Architecture docs: `docs/architecture/` (restructure, rename, remove bloat)
- Kanban: `docs/project/kanban.md` and `data/projects.json` (SWOT/risk sections)
- Do NOT modify any Python, Rust, or TypeScript source code
