## Problem Statement

The workspace has five meta-quality problems:

1. **Agent file scatter**: agent definitions live in `.github/agents/`, but memory and log files live in `docs/agents/`. Agents, IDE tooling, and the backend API all reference either location ΓÇõ creating cognitive overhead and misrouted references.

2. **No docstrings mandate**: `@5test` and `@6code` write functions without Google-style docstrings. There is no enforcement hook (no ruff `D` rules configured for agent-written code). Technical debt accumulates silently.

3. **Bloated architecture docs**: `docs/architecture/` has 15 files. Seven (`Code.md`, `Design.md`, `Exec.md`, `Git.md`, `Plan.md`, `Ql.md`, `Think.md`) are near-empty stubs. `Architecture.generated.md` is a generated artefact checked in to source control. `todolist.md` is a stale agenda. The signal-to-noise ratio is low.

4. **No risk / SWOT tracking**: `docs/project/kanban.md` has 7 lanes but no SWOT analysis or risk register. Project risks are tracked ad-hoc (or not at all).

5. **Shallow @2think output**: the @2think agent has 4 steps (Understand, Generate options, Compare, Handoff). It lacks explicit research task types (literature review, alternative enumeration, prior-art search, constraint mapping, stakeholder impact) and has no minimum-depth guideline, so agents often proceed to design without adequate exploration.

## Options Explored

### Sub-goal 1 ΓÇõ Agent File Co-location

**Option A (Recommended): Move memory/log files + update all references**
- Move all `docs/agents/*.memory.md` and `docs/agents/*.log.md` to `.github/agents/`
- Update `backend/app.py` line 60: `_LOGS_DIR` path constant (one-liner, no logic change)
- Update `tests/test_agent_doc_frequency.py`: `DOCS_DIR` constant
- Update `tests/docs/test_agent_workflow_policy_docs.py`: path string assertions
- Update `scripts/AgentDocFrequency.py`: default `--docs-dir` value
- Update all `.github/agents/*.agent.md` references from `docs/agents/` ΓÁÈ `.github/agents/`
- Pros: single-source-of-truth for all agent files; no split brain; backend already uses `.github/agents` for `.agent.md`
- Cons: touches Python test/script files; those path strings are config, not logic ΓÇõ risk is low

**Option B: Copy only (keep originals in docs/agents/)**
- Symlink or duplicate memory/log files into `.github/agents/`
- Pros: zero Python changes
- Cons: double maintenance; misleads agents about canonical location; git symlinks are unreliable on Windows

**Option C: Keep current layout**
- Do nothing for sub-goal 1
- Pros: no risk
- Cons: split-brain persists; copilot-instructions.md already directs agents to `docs/agents/`

ΓÁÈ **Recommended: Option A** ΓÇõ the diff is small (path strings only) and achieves the acceptance criteria cleanly.
