## Test Strategy

### Scope

The hook runs in a browser environment; full DOM/WebSocket integration tests
require jsdom + complex mocking.  For this feature the highest-value tests are:

1. **Structural** — verify the module exports the expected symbols and that the
   `UseWebSocketOptions` interface carries all documented fields.
2. **Formula unit tests** — verify the backoff formula and jitter bounds
   independently of React/DOM, using pure `Math.*` assertions.
3. **Backwards-compat** — verify the `reconnectDelay → baseDelay` alias logic.

DOM-integrated reconnect-cycle tests (measuring actual timer fire order) are
deferred to a future prj that sets up jsdom + fake timers.
