## Project Overview

Add exponential backoff with jitter to the `useWebSocket` React hook so that frontend
WebSocket disconnects are handled gracefully. Retry delays grow from ~1 s to a 30 s cap
with ±50% random jitter to prevent thundering-herd reconnect storms.

## Goal & Scope

**Goal:** Replace the fixed-delay reconnect in `useWebSocket` with configurable exponential
backoff and jitter; expose `reconnectAttempts` and `maxRetries` to callers.

**In scope:**
- `web/hooks/useWebSocket.ts` — add `baseDelay`, `maxDelay`, `maxRetries` options; backoff logic; `reconnectAttempts` return value; backwards-compatible `reconnectDelay` deprecation
- `web/hooks/useWebSocket.test.ts` — comprehensive Vitest tests: exports, backoff formula, jitter bounds, backwards-compat defaults
- 9 project doc artifacts

**Out of scope:**
- Backend changes
- Other frontend files
- Changes to `AiPanel.tsx`, `VideoPanel.tsx`, or `AgentChat.tsx` (they consume the hook via its stable interface)

## Acceptance Criteria

- Reconnect delay grows exponentially (doubles each attempt) up to `maxDelay` (default 30 000 ms)
- Jitter of ±50% applied: actual delay is in `[0.5 * exp, exp]`
- `maxRetries` option stops reconnection after N failed attempts
- `reconnectAttempts` counter exposed in hook return value
- `connected` resets to `false` on close; `reconnectAttempts` resets to 0 on successful open
- All 28 Vitest tests pass; Python pytest suite still at 835+, 0 warnings
