### Specific Checks

| Check | Finding |
|---|---|
| `Math.random()` used for jitter — not cryptographic | ✅ Acceptable — jitter is not a security primitive; CSPRNG is not required for retry spreading |
| `JSON.parse` in `ws.onmessage` | ✅ Wrapped in try/catch — malformed messages silently dropped; no crash |
| Timer leak on unmount | ✅ `clearTimeout` in cleanup; `active` flag stops reconnect loop |
| XSS via `event.data` | ✅ Data is parsed to a typed `WsMessage` object; not rendered as HTML |
| Infinite reconnect resource consumption | ✅ `maxRetries` option available; default is `Infinity` (documented) — caller's responsibility |
