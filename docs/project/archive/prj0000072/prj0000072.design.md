## Interface Changes

### `UseWebSocketOptions` (extended, backwards-compatible)

```typescript
export interface UseWebSocketOptions {
  onMessage?: (msg: WsMessage) => void;
  /** @deprecated Use baseDelay instead. Kept for backwards compatibility. */
  reconnectDelay?: number;
  /** Initial reconnect delay in ms before jitter (default: 1000). */
  baseDelay?: number;
  /** Maximum reconnect delay cap in ms (default: 30000). */
  maxDelay?: number;
  /** Maximum number of reconnect attempts before giving up (default: Infinity). */
  maxRetries?: number;
}
```

**Backwards-compat rule:** if only `reconnectDelay` is supplied, it maps to
`baseDelay` via `baseDelay = reconnectDelay ?? 1000`.  Existing callers pass-
through transparently.

### Hook return type (extended)

```typescript
return { send, connected, reconnectAttempts };
//                        ^^^^^^^^^^^^^^^^^ тАФ new
```

`reconnectAttempts` starts at 0, increments for each failed-close/scheduled-
reconnect cycle, and resets to 0 on a successful open.

## Core Logic

```
attempt = 0
on close:
  if attempt >= maxRetries тЖТ stop
  exp     = min(maxDelay, baseDelay ├Ч 2^attempt)
  jitter  = exp ├Ч (0.5 + random() ├Ч 0.5)   # full-jitter in [0.5┬╖exp, exp]
  attempt = attempt + 1
  setReconnectAttempts(attempt)
  setTimeout(connect, jitter)

on open:
  attempt = 0
  setReconnectAttempts(0)
  setConnected(true)
```

## Constraints

1. **No new dependencies** тАФ pure React + TypeScript changes only.
2. **No breaking changes to existing callers** тАФ `reconnectDelay` preserved as alias.
3. **File scope:** `web/hooks/useWebSocket.ts` only.
4. **No DOM-required tests** тАФ all unit tests are pure-JS formula checks; no jsdom.
5. **Stale closure prevention** тАФ all options stored in `useRef` so the effect
   never captures outdated closures across re-renders.
