# prj0000030 — Agent Doc Frequency: Git History

## Branch
`prj0000030-agent-doc-frequency`

## What Was Done
Implemented the agent documentation frequency analyser — a tool that scans `docs/agents/*.memory.md` files and reports how recently and how often each has been updated.

### Script created
`scripts/AgentDocFrequency.py`:
- `DocStats` dataclass: name, path, commit_count, last_updated, staleness (0.0–1.0)
- `analyse_docs(docs_dir, repo_root, days_stale)` — queries `git log` per file, computes staleness score
- `format_table(stats)` — renders Markdown pipe table sorted by staleness descending
- `main()` CLI — `--docs-dir`, `--days-stale` arguments; prints table + summary counts

**Staleness formula:** `min(1.0, days_since_last_commit / days_stale)`

### Tests created
`tests/test_agent_doc_frequency.py` — 12 tests:
- **Unit (DocStats)**: default values, field assignability
- **Unit (format_table)**: header, separator, file name, "never" for no history
- **Integration (analyse_docs)**: all 10 memory files found, sorted by staleness desc, scores in [0,1], non-negative commit counts, full table output
- **CLI smoke test**: `main()` runs without error
