# Project prj000104: Requirements Deduplication

## Vision
Audit and consolidate duplicate dependencies across multiple requirements files to maintain a single source of truth for dependency management in PyAgent.

## Goals
1. **Audit dependencies**: Scan all requirements*.txt files
2. **Identify duplicates**: Find overlapping dependencies
3. **Create consolidation strategy**: Define dependency hierarchy
4. **Add CI/CD check**: Prevent duplicate dependencies
5. **Document dependency strategy**: Create dependency management guide

## Scope
- Analyze `requirements.txt`, `requirements-ci.txt`, `backend/requirements.txt`
- Create deduplication script
- Add CI/CD check for duplicates
- Document dependency consolidation strategy
- Create reference mapping for all dependencies

## Success Criteria
- ✅ All duplicate dependencies identified
- ✅ Consolidation strategy documented
- ✅ CI/CD check prevents new duplicates
- ✅ Dependency mapping updated
- ✅ Documentation covers dependency sources

## References
- **Requirements Files**: `requirements.txt`, `requirements-ci.txt`, `backend/requirements.txt`
- **Config**: `pyproject.toml` [project.dependencies]
- **CI/CD**: `.github/workflows/` for dependency checks
