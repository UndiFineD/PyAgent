## Project Overview

The GitHub Actions CI directory had grown to 5 workflow files despite the intent to keep CI
minimal and rely on pre-commit for local-first testing. This project removes the 4 redundant
workflows, adds a new `security.yml` for CodeQL scanning (addressing a security gap), and
documents the local testing workflow.

## Goal & Scope

**Goal:** Reduce workflow files from 5 to 2 (`ci.yml` + `security.yml`); wire up CodeQL scanning
that was previously missing; document local testing in `docs/setup.md`.

**In scope:**
- `.github/workflows/` — delete 4 redundant files, create `security.yml`
- `docs/setup.md` — add Local Testing section
- `tests/ci/test_workflow_count.py` — new test asserting exactly 2 workflow files
- `tests/structure/test_ci_yaml.py` — add setup.md assertion
- `data/projects.json` + `docs/project/kanban.md` — lane updates

**Out of scope:**
- Changes to `.pre-commit-config.yaml` (already covers ruff + mypy)
- Changes to Python/Rust/TS source code
- `core validate()` API implementation (separate project)
