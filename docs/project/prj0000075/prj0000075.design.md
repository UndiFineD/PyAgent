## Interface & Constraint Decisions

### Workflow files after this project

| File | Action | Rationale |
|------|--------|-----------|
| `.github/workflows/ci.yml` | **Keep unchanged** | Sole functional-correctness gate; satisfies all test assertions |
| `.github/workflows/security.yml` | **Create new** | Wires CodeQL for Python + JavaScript; addresses zero-security-gate gap |
| `.github/workflows/core-quality.yml` | **Delete** | Broken (references `validate()` that doesn't exist); 100% overlaps ci.yml |
| `.github/workflows/pm.yml` | **Delete** | PM tests in ci.yml shard 3; template generation is not a quality gate |
| `.github/workflows/quality.yml` | **Delete** | ruff + mypy 100% duplicated by pre-commit; Python 3.12 version mismatch |
| `.github/workflows/testing-infra.yml` | **Delete** | Structure tests in ci.yml shard 1; full-suite run duplicates ci.yml |

### `security.yml` design

- **Trigger:** `push` to `main`; `pull_request` to `main`; `schedule` weekly (Monday 07:00 UTC)
- **Languages:** `python`, `javascript` ‚Äî matching the custom query packs in repo root
- **Custom queries:** Reference `codeql-custom-queries-python/` and `codeql-custom-queries-javascript/`
- **Permissions (minimal):**
  - `contents: read`
  - `security-events: write` (required to upload SARIF to GitHub Security tab)
  - `actions: read` (required for CodeQL init on private repos)
- **Jobs:** Single `analyze` job with a `matrix.language` dimension (`[python, javascript]`)
- **Steps:** `checkout` ‚Üí `codeql/init` (with custom queries) ‚Üí `codeql/autobuild` ‚Üí `codeql/analyze`
- **Action versions:** `github/codeql-action/init@v3`, `autobuild@v3`, `analyze@v3`

### `docs/setup.md` additions

A **"Local Testing"** section is appended documenting:
1. Pre-commit hooks (ruff, mypy, enforce-branch) ‚Äî when/how to run
2. `pytest` ‚Äî full suite + shard equivalents
3. Local CodeQL: `C:\Dev\CodeQL` CLI location for developers who want to run queries locally

### New test: `tests/ci/test_workflow_count.py`

Asserts exactly **2** workflow files exist (`ci.yml` and `security.yml`) and no others.
This replaces the hypothetical 1-file assertion from the think.md recommendation (updated for the
security.yml addition).
