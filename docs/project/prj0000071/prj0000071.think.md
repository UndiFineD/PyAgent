# prj0000071 — Dark Mode Accessibility: Think

_Agent: @2think | Date: 2026-03-25 | Branch: prj0000071-dark-mode-accessibility_

## Problem Statement

The NebulaOS dark theme UI has several confirmed WCAG 2.1 Level AA accessibility violations
across five component files. These failures affect keyboard-only users, screen reader users,
and users with contrast sensitivity needs.

---

## Confirmed Violations

### 1. Icon-only buttons with no accessible name (WCAG 4.1.2 — Level A)

**File: `web/components/Window.tsx` lines 252–260**
- Minimize button: renders `<Minus>` icon only, no `aria-label`
- Maximize/Restore button: renders `<Maximize2 />` / `<Square />` icon only, no `aria-label`,
  no state communicated (`aria-pressed` / `aria-expanded`)
- Close button: renders `<X>` icon only, no `aria-label`
- Menu button (~line 200): renders `<Menu>` icon only, no `aria-label`

**File: `web/apps/Paint.tsx` lines 73–105**  
- Pencil tool button: `<Pencil>` icon only, no `aria-label`
- Eraser tool button: `<Eraser>` icon only, no `aria-label`
- Clear canvas button: `<Trash2>` icon only, no `aria-label`

### 2. Interactive non-semantic elements with no keyboard interaction (WCAG 1.3.1, 4.1.2 — Level A)

**File: `web/apps/ProjectManager.tsx` line 274**
- `<div onClick>` with `draggable` — expands project card on click but:
  - No `role="button"` or equivalent
  - No `tabIndex`
  - No `onKeyDown` for Enter/Space
  - Screen readers and keyboard users cannot activate the expand

### 3. Missing visible focus indicator (WCAG 2.4.7 — Level AA)

**File: `web/apps/Editor.tsx` line 38**: `focus:outline-none` with no alternative
**File: `web/App.tsx` line 371**: `focus:outline-none` on `role="switch"` with no alternative

### 4. Form controls without labels (WCAG 1.3.1 — Level A)

**File: `web/apps/Paint.tsx` lines 90–101**
- `<input type="color">` — no `<label>` and no `aria-label`
- `<input type="range">` — no `<label>` and no `aria-label`

### 5. Colour contrast failure (WCAG 1.4.3 — Level AA)

**File: `web/styles/themes.css` line 5**
- `--color-accent: #e94560` on `--color-bg: #1a1a2e`
- Computed contrast: **4.14:1** — below the 4.5:1 threshold for normal (< 18pt) text
- Fix: raise to `#f25e77` → computed contrast **5.01:1** (passes AA for normal text)

---

## Alternatives Considered

### Option A — Add `aria-label` props only (no visual changes)
- **Pro:** minimal diff, no visual regression risk
- **Con:** misses focus indicator failures (users who rely on focus ring cannot navigate)

### Option B — Full WCAG AAA remediation
- **Pro:** highest accessibility bar
- **Con:** budget tier S; AAA contrast is 7:1 — would require major theme redesign

### Option C — Fix confirmed AA violations only (chosen)
- Fixes all 5 violation categories with minimum necessary changes
- No visual redesign — aria-labels are invisible; focus rings are subtle; contrast delta is small
- Passes WCAG 2.1 AA

**Decision: Option C.**
