## Project Overview

WCAG 2.1 AA accessibility audit and remediation for the NebulaOS dark theme. The audit
identified five categories of failures spanning icon-only buttons without accessible names,
missing keyboard interaction on interactive `<div>` elements, removed focus indicators,
missing form labels, and a colour-contrast shortfall in the accent colour token.

## Goal & Scope

**Goal:** Bring the NebulaOS UI to WCAG 2.1 Level AA compliance for the dark theme by fixing
all confirmed A and AA violations found during the audit.

**In scope:**
- `web/components/Window.tsx` — aria-labels on Minimize / Maximize / Close / Menu buttons; focus rings
- `web/apps/Paint.tsx` — aria-labels on Pencil / Eraser / Clear buttons; labels on color and range inputs
- `web/apps/Editor.tsx` — restore visible focus ring on textarea
- `web/App.tsx` — restore visible focus ring on taskbar toggle switch
- `web/apps/ProjectManager.tsx` — keyboard (Enter/Space) handler + `role="button"` on project card
- `web/styles/themes.css` — raise `--color-accent` to pass ≥ 4.5:1 contrast against `--color-bg`
- `tests/web/test_a11y_checklist.py` — structural accessibility checklist test
- 9 workflow artifacts

**Out of scope:**
- Automated tools (axe, Playwright) — out of scope for budget tier S
- Non-dark theme accessibility work (light theme considered already compliant)
- Screen reader testing
- Backend changes
