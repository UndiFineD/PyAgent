_Agent: @3design | Date: 2026-03-25 | Branch: prj0000071-dark-mode-accessibility_

## Selected Approach

Fix all confirmed WCAG 2.1 AA violations with the minimum necessary changes per file.
No new components or abstractions. All changes are additive (aria attributes, Tailwind classes).

---

## Per-File Design

### `web/components/Window.tsx`

**Problem:** 4 icon-only buttons have no accessible name; buttons lack focus ring.

**Fix:**
- Add `aria-label="Minimize"` to Minimize button
- Add `aria-label` dynamically: `aria-label={window.isMaximized ? "Restore" : "Maximize"}`
  to communicate toggle state
- Add `aria-label="Close"` to Close button
- Add `aria-label="Window menu"` to Menu button
- Add `focus-visible:ring-1 focus-visible:ring-white/60` to all four buttons
  (using `focus-visible:` to not show on mouse click, only keyboard focus)

### `web/apps/Paint.tsx`

**Problem:** 3 icon-only tool buttons; 2 unlabelled inputs.

**Fix:**
- Add `aria-label="Pencil"` to pencil button, `aria-label="Eraser"` to eraser button
- Add `aria-label="Clear canvas"` to trash button
- Add `aria-label="Stroke colour"` to `<input type="color">`
- Wrap range input in a `<label>` with hidden text: add `aria-label="Stroke width"` directly

### `web/apps/Editor.tsx`

**Problem:** `focus:outline-none` removes focus ring from full-screen textarea.

**Fix:**
- Replace `focus:outline-none` with `focus:outline-none focus:ring-2 focus:ring-os-accent`
  (the ring-2 provides a visible indicator; outline-none prevents the default browser
  double-ring artefact)

### `web/App.tsx`

**Problem:** Toggle switch button has `focus:outline-none` — removes focus ring.

**Fix:**
- Replace `focus:outline-none` with `focus:outline-none focus:ring-2 focus:ring-blue-400`
  (matches the switch's active colour `bg-blue-500` for consistency)

### `web/apps/ProjectManager.tsx`

**Problem:** `<div onClick>` with no role, no tabIndex, no keyboard handler.

**Fix:**
- Add `role="button"` and `tabIndex={0}` to the card `<div>`
- Add `onKeyDown` handler: activate expand on `Enter` and `Space` keys
- Add `aria-expanded={expanded}` to communicate state to screen readers

### `web/styles/themes.css`

**Problem:** `--color-accent: #e94560` gives 4.14:1 contrast on `--color-bg: #1a1a2e` — fails 4.5:1 AA.

**Fix:**
- Change to `--color-accent: #f25e77`
  - Computed luminance: 0.2819
  - Contrast against `--color-bg` (#1a1a2e, lum ≈ 0.01633): **5.01:1** ✓
  - Visual difference: subtle brightening of 13 units on the red channel; maintains the
    red/rose identity of the NebulaOS palette

## Focus Ring Strategy

Use `focus-visible:` pseudo-class (Tailwind v3+) for interactive elements that are
also mouse-clicked (buttons), to avoid showing a ring on mouse press while still
serving keyboard users. Use `focus:` for pure keyboard-input elements (textareas,
toggles) where a persistent ring is acceptable and expected.
