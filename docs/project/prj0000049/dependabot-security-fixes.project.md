## Project Overview

Resolve all active Dependabot security alerts on the PyAgent repository. As of 2026-03-23, GitHub reported 6 vulnerabilities (1 high, 3 moderate, 2 low) across the Python, Rust, and JavaScript dependency ecosystems.

## Goal & Scope

**Goal:** Eliminate all active Dependabot security vulnerabilities by bumping affected dependency versions to patched releases.

**In scope:**
- Bump dependency versions in `requirements.txt`, `backend/requirements.txt`, `rust_core/Cargo.toml`, `web/package.json` (and related lock files)
- Update `pyproject.toml` if a Python dep is pinned there
- Update `Cargo.lock` as generated by `cargo update`

**Out of scope:**
- Changes to application source code beyond what a version bump requires
- Changes to CI workflow files unless a newly required step is part of the fix
- Architectural refactors or feature additions

## Branch Plan

**Expected branch:** `prj0000049-dependabot-security-fixes`  
**Base:** `main`  
**Scope boundary:** Dependency version pins only — no application logic changes permitted.  
**Handoff rule:** After all bumps pass CI and `cargo audit` reports 0 vulnerabilities, hand off to @8ql for security review, then @9git for merge.  
**Failure rule:** If a dependency bump causes test failures that cannot be fixed by a minor patch-level adjustment, the bump is reverted and the finding is documented in `dependabot-security-fixes.ql.md` as an accepted risk with rationale.
