## Project Overview

Full workspace audit to surface future improvement ideas for the PyAgent codebase. This project
systematically surveys every significant area of the workspace — source code (`src/`, `rust_core/`,
`backend/`, `web/`), tests, documentation, CI configuration, agent definitions, and infrastructure
— and produces a structured SWOT-prioritised table of at least 20 future improvement ideas to be
appended to `docs/project/kanban.md` as a new `## Future Ideas` section.

**No source code is modified by this project.** Each surfaced idea becomes an independent future
`prjNNNNNNN` entry; this project only identifies and prioritises them.

## Goal & Scope

**Goal:** Produce a SWOT-prioritised idea table in `docs/project/kanban.md` with >= 20 distinct
future improvement ideas spanning at least 6 distinct areas of the codebase, each carrying
Priority (P1-P4), Impact (H/M/L), Urgency (H/M/L), and a 1-2 sentence description.

**In scope:**
- `docs/project/prj0000076/` — all project artifacts (think, design, plan, test, code, exec, ql, git stubs)
- `docs/project/kanban.md` — new `## Future Ideas` section appended (no modification to the existing project table)
- `data/projects.json` — lane/status updates for prj0000076 only

**Out of scope:**
- Implementing any of the surfaced ideas (each becomes its own future `prjNNNNNNN`)
- Modifying Python / Rust / TypeScript source code
- Adding new rows to the existing project table in `kanban.md`
- Creating new project folders for the surfaced ideas
- Raising PRs for individual ideas
