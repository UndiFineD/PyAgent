# prj0000076 — future-ideas-kanban

## Goal
Systematically survey every significant area of the PyAgent workspace — source code, tests, documentation, CI, architecture, agent definitions — and produce a structured table of future improvement ideas. This table should be added to `docs/project/kanban.md` as a new section ("Future Ideas") with the following columns: ID, Name, Priority (P1–P4), Impact (H/M/L), Urgency (H/M/L), Description.

Sources to review:
- `src/` — Python agent and tool implementations
- `rust_core/` — Rust extension (FFI surface, coverage, performance opportunities)
- `backend/` — FastAPI backend (coverage, security, missing endpoints)
- `web/` — React/Vite frontend (UX gaps, missing features)
- `tests/` — test coverage gaps, flaky tests, missing integration scenarios
- `docs/` — missing documentation, outdated content
- `.github/` — agent definitions, workflow gaps
- `data/` and `deploy/` — data model improvements, deployment improvements
- `pyproject.toml`, `requirements.txt`, `requirements-ci.txt` — dependency hygiene

## Scope boundary
- Output: a new `## Future Ideas` section in `docs/project/kanban.md`
- Output: at minimum 20 distinct ideas, each with Priority, Impact, Urgency, and a 1–2 sentence description
- Do NOT modify any Python/Rust/TS source code
- Do NOT create new project folders for the ideas (that happens later, one by one)
- Do NOT raise PRs for the individual ideas
